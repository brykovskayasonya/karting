# BUG-001: Превью маршрута показывает синее «море» вместо бежевой «земли»

## Симптом

На экране маршрута (`RouteMapSheet` → `BS-004-route-map`) превью карты рисовало
статичную заглушку с синим прямоугольником («море») и зелёным берегом сбоку —
оформление, оставшееся от прежней (лодочной/SUP) темы. Для картинг-центра это
не подходит: нужна заглушка с бежевой «землёй» и зелёными «газонами» вместо моря.

Правка кода не давала видимого эффекта в приложении на устройстве даже после
нескольких пересборок — картинка оставалась прежней.

## Требования

- `01-analysis/3-design-brief/BS-004-route-map.md` — заглушка карты маршрута:
  статичный мок-скриншот, не зависящий от координат/БД, с CTA «Проложить маршрут»
  и «Открыть в картах».
- Задача: заменить цветовую схему заглушки (убрать морскую тематику).

## Причина / диагностика

1. Файл `RouteMapPreviewFallback.kt` — это **не настоящая карта**, а Compose
   `Canvas`, целиком нарисованный в коде (не зависит от БД/координат), поэтому
   смена данных маршрута/точки встречи не могла повлиять на картинку — нужно
   было менять именно код отрисовки.
2. Путь файла:
   `shared/src/commonMain/kotlin/com/volna/app/map/RouteMapPreviewFallback.kt`
3. После правки (замена `waterColor`/`drawRoundRect(waterColor)` на бежевую
   `landColor` без отрисовки «моря») изменения были корректно сохранены и
   присутствовали в файле — это подтверждено сверкой содержимого файла.
4. Тем не менее в приложении на физическом устройстве
   изменения не отображались. Диагностика показала:
   - Build (`:androidApp:assembleDebug`) проходил успешно, `shared` модуль
     пересобирался с новым кодом (`shared:compileDebugKotlinAndroid`
     выполнялся, а не `UP-TO-DATE`).
   - В логе **Run** при этом не было строки `Installing APK...` — Android
     Studio при повторном запуске просто заново активировала уже установленную
     на устройстве Activity (`adb shell am start ...`), **не переустанавливая
     APK** («Delta Install» посчитал переустановку не нужной).
   - Ручное удаление приложения (`adb uninstall com.volna.app` → `Success`)
     и повторный `Run` устранили проблему: APK был установлен заново, и
     заглушка карты обновилась.

## Исправление

- В `RouteMapPreviewFallback.kt`: убрана переменная `waterColor` и её
  `drawRoundRect`, весь фон канвы теперь рисуется как `landColor`
  (`0xFFF6F3ED`, бежевый) с газонами `parkColor` (`0xFFD5F3BA`), без «моря».
- Обходной путь для среды разработки: для гарантированной установки свежей
  сборки на устройстве — либо `adb uninstall <applicationId>` перед `Run`,
  либо включить в Android Studio настройку
  **Settings → Build, Execution, Deployment → Deployment →
  "Always install with package manager (disable delta install)"**.


## Проверка

Проверено вручную: содержимое файла `RouteMapPreviewFallback.kt` сверено и
подтверждено (новый код без `waterColor`), выполнено `adb uninstall
com.volna.app` (результат `Success`), после повторного запуска через Android
Studio APK установился заново. Причина, по которой правки не отображались
(Delta Install переиспользовал старую версию приложения на устройстве),
найдена и устранена.

## Коммит

Коммит с правкой `RouteMapPreviewFallback.kt` и текущими доками добавлен в
рамках финального push перед сдачей задания.
